适用于多种前端框架的库，用于检测网站是否存在新版本，让开发者可以告知用户。

`use-upgrade` 可以零配置、零侵入的开箱即用，但同时也提供了丰富的配置项，几乎所有行为都可定制；使用它不会给网络带来额外负担，因为在多个标签页或浏览器处于后台等场景时，它会自动暂停网络活动；同时，它还提供了 React Hooks，使开发相关 UI 交互变得更为容易。

`use-upgrade` 目前已确定兼容的生态：

- **React 生态：** Vite、Create-React-App、Next.js、Umi.js、Taro
- **Vue 生态：** Vite、Vue CLI、Nuxt.js
- Angular

*（以上列出的框架均可兼容，但小部分框架仍需少许配置；未来的 1.0 正式版将完全零配置适配以上所有框架）*

此外，还可以选用配套的 `use-upgrade-webpack-plugin`，它可以与构建工具集成，在无需修改代码的情况下控制某版本是否触发 `use-upgrade`，使得小型版本发布可以避免打扰用户。

> `use-upgrade` 的 1.0 正式版即将发布，正式版无需任何配置即可兼容更多框架，和 Next.js 等 SSR 框架集成也不会出现问题，同时额外提供 Vue Hooks，代码仓库中还将附带测试用例；<br />还会同步发布配套的 `vite-plugin-use-upgrade` 以适配 Vite 工具链。
